# AniSync 产品需求文档 (PRD)

## 1. 产品概述

AniSync 是一款本地运行的追番记录软件，帮助用户管理自己的动漫观看记录、追踪观看进度、查看当季新番放送日历。

### 1.1 产品定位

- 本地单用户应用（MVP 阶段）
- 前后端分离架构
- 数据存储在本地 SQLite

### 1.2 目标用户

- 需要管理个人追番记录的动漫爱好者

---

## 2. 功能需求

### 2.1 追番记录管理（核心）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 添加番剧 | 手动添加或通过搜索自动获取番剧信息 | P0 |
| 编辑番剧 | 修改番剧信息、状态、评分 | P0 |
| 删除番剧 | 从记录中移除番剧 | P0 |
| 观看状态 | 支持三种状态：想看 (plan) / 在看 (watching) / 看完 (completed) | P0 |
| 进度记录 | 记录当前观看集数、总集数，支持一键 +1 集 | P0 |
| 列表展示 | 按状态分类展示所有番剧 | P0 |
| 搜索筛选 | 按名称搜索、按状态/评分筛选 | P1 |
| 评分系统 | 1-10 分评分 | P1 |

### 2.2 自动获取番剧信息

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 搜索番剧 | 输入关键词搜索番剧库 | P0 |
| 自动填充 | 获取封面图、简介、集数、放送日期等信息 | P0 |
| 数据源 | 调用 Bangumi API（bgm.tv，中文搜索体验优秀） | P0 |

### 2.3 观看进度追踪

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 记录集数 | 手动更新当前看到第几集 | P0 |
| 进度显示 | 显示进度条和百分比（当前集数/总集数） | P0 |
| 批量更新 | 快速将进度更新到最新集数 | P1 |

### 2.4 放送日历

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 当季新番 | 按星期几展示当前季度的放送时间表 | P1 |
| 日历视图 | 周视图，每天展示该日放送的番剧 | P1 |

### 2.5 数据导入导出

| 功能 | 描述 | 优先级 |
|------|------|--------|
| JSON 导出 | 导出全部数据为 JSON 文件 | P1 |
| JSON 导入 | 从 JSON 文件导入数据 | P1 |
| CSV 导出 | 导出为 CSV 格式，兼容 Excel | P2 |
| CSV 导入 | 从 CSV 文件导入数据 | P2 |
| Bangumi 格式 | 兼容 Bangumi 导入导出格式 | P2 |

---

## 3. 非功能需求

### 3.1 性能

- 页面加载时间 < 2s
- 番剧搜索响应时间 < 3s（含网络请求）

### 3.2 数据安全

- SQLite 数据库文件存储在本地 `data/` 目录
- 支持数据备份（通过导出功能）

### 3.3 兼容性

- 支持 Chrome / Edge / Firefox 最新版本
- 响应式设计，适配桌面端（1280px+）

---

## 4. MVP 范围（第一版）

**包含功能：**
- 追番 CRUD（增删改查）
- 观看状态管理（想看/在看/看完）
- 观看进度追踪
- 番剧搜索（Bangumi API）+ 自动填充
- 列表展示 + 搜索筛选

**暂不包含：**
- 用户登录系统（后续版本）
- 放送日历（第二版）
- 数据导入导出（第二版）
- 评分系统（第二版）
- 统计图表（第三版）

---

## 5. 页面结构

```
首页（番剧列表）
├── 搜索栏（本地筛选）
├── 状态筛选标签（全部/想看/在看/看完）
├── 番剧卡片列表
│   ├── 封面图
│   ├── 番剧名称
│   ├── 观看状态
│   ├── 进度（x/y 集）
│   └── 评分
└── 添加番剧按钮

添加/编辑番剧弹窗
├── 番剧搜索框（Bangumi API）
├── 搜索结果列表（点击自动填充）
├── 番剧信息表单
│   ├── 名称（必填）
│   ├── 封面图 URL
│   ├── 总集数
│   ├── 当前进度
│   ├── 观看状态
│   └── 评分
└── 保存/取消按钮
```
